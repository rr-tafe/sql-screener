1 · Getting started

1
Open the file in a browser. Double-click the .html file your screener sent you. It will open in your default browser. No internet connection is needed — the entire tool, database, and SQL engine are self-contained inside that one file.
2
Enter your full name and click Begin Test. The timer starts the moment you click Begin.
3
Do not close or refresh the tab during the assessment. Refreshing reloads the page and clears your in-progress work.
Important: Once you click "Finish Work" and confirm, your results download immediately. Make sure you are done before you click it.

2 · What the tool looks like

These screenshots show exactly what you will see when you open the assessment file. Review them now so the interface is familiar when the file arrives 5 minutes before your session.

Start screen

When you open the file, you will see a card in the centre of the screen. Enter your full name and click Begin Test. The timer starts immediately.

Main interface

After you begin, the three-panel layout appears: the sidebar on the left (timer, question nav, Finish Work button), the questions panel in the centre, and the playground panel on the right (schema reference, notes, scratch SQL editor).

A question with a result

Each question shows the prompt, the expected columns, a SQL editor, and Run / Format buttons. After you run a query, a status badge (PASS, FAIL, or ERROR) appears below the buttons, followed by the result table.

Playground panel

The right panel is your scratch space. The schema reference at the top lists all tables and columns. Below that is a free-text notes area, then an ungraded SQL editor for exploration, and collapsible sections for SQLite Reference and Query History.

Resizing panels

You can drag the vertical dividers between panels to resize them. Drag the divider between the sidebar and center panel, or the one between the center and right panel. This is useful if you want more space for writing SQL.

Collapsing the sidebar

Click the ‹ button at the top of the sidebar to collapse it and gain more screen space. Click › to expand it again. The question nav dots and timer remain visible even when collapsed.

Light / dark mode

Click the ☀ button in the top-right area of the right panel to switch between light and dark themes. All editors change together.

4 · Answering questions

Each question in the center panel contains:

  • Question prompt — the business question you need to answer with SQL.
  • Expected columns — the column names your result set must include, in order.
  • SQL editor — where you write your answer.
  • Run and Format buttons.
  • Status badge — appears after you run your query.
  • Results table — the rows your query returned.
Use the sidebar navigation links to jump directly to any question. Each link has a coloured dot showing that question's current status.

Running a query

Write your SQL in the editor for that question and click Run. The query executes against a real SQLite database embedded in the tool. Results appear immediately below the editor, and a status badge shows whether your answer is correct.

Status badges

BadgeMeaning
PASS Your result set matches the expected answer. Well done.
FAIL Your query ran without errors but the result did not match. Review your logic and try again.
ERROR Your query has a syntax or runtime error. The error message is shown in the badge. Fix the SQL and re-run.
NOT PERMITTED Your query was blocked by the SQL guard (see Section 7). Only SELECT queries are allowed for graded questions.

You can re-run a question as many times as you like. The status badge always reflects your most recent run.

5 · Playground & tools

The right panel is your scratch space. Nothing you do there affects your graded score.

SQL Playground

Use the playground SQL editor to explore the database freely — run exploratory queries, check what's in each table, test intermediate logic. Click Run or use Shift+Alt+F to format. Queries run here appear in Query History for easy reuse.

Schema Reference

At the top of the right panel, all table names and their columns are listed. Click the ▲ button next to "Playground" to collapse or expand this panel.

Notes

A free-text notes area sits below the schema reference. Use it to jot down intermediate calculations, working thoughts, or column relationships you notice. Your notes are included in the results report.

SQLite Function Reference

Click SQLite Reference in the right panel to expand a searchable reference of built-in SQLite functions: aggregate, string, numeric, date/time, conditional, window, and type conversion. Search by function name or description.

Query History

All successful playground queries are saved automatically. Click Query History to expand the list. Click the ▶ play button next to any past query to load it back into the playground editor and re-run it.

6 · How grading works

Grading is automatic and happens the moment you click Run on a graded question. You do not need to do anything special to submit an answer — running the query is the submission.

What the grader checks

  • Values only — column names in your output do not need to match exactly; only the values in the positions specified by "Expected columns" matter.
  • Row order is ignored — your rows can come back in any order and still pass.
  • NULL handling — NULLs are treated as a distinct value and must appear where expected.
Practical tip: If you are confident your values are correct but you are getting FAIL, check that you are selecting the columns in the order shown under "Expected columns", and that you are not accidentally including extra columns.

Editor features

  • Autocomplete — The editor knows the schema. Start typing a table or column name and press Ctrl+Space to trigger suggestions, or just keep typing and suggestions will appear automatically.
  • Syntax highlighting — SQL keywords, strings, and identifiers are colour-coded.
  • Line numbers — Displayed on the left side of each editor.
  • Bracket matching — Opening brackets are automatically matched and parentheses are closed for you.
  • Line wrapping — Long lines wrap within the editor so you do not need to scroll horizontally.

8 · SQL rules

To protect the integrity of the embedded database, a small set of SQL statements is restricted.

AllowedBlocked
SELECT … — any SELECT query
CREATE TEMP TABLE … — temporary tables for intermediate work 		INSERT, UPDATE, DELETE, DROP, ALTER, CREATE TABLE (non-temporary), and any other data-modifying statement

If you run a blocked statement you will see a NOT PERMITTED badge. The database is unaffected. Simply change your approach to use a SELECT query instead.

Need a temporary working table? Use CREATE TEMP TABLE my_cte AS SELECT … — this is explicitly allowed. CTEs (WITH … AS (SELECT …)) inside a SELECT are also fully supported and often cleaner.

9 · Finishing the assessment

1
When you are ready to submit, click the Finish Work button at the bottom of the left sidebar.
2
A confirmation dialog will appear. Read it carefully. Click OK only when you are sure you are done — you cannot undo this.
3
A results file will download automatically to your browser's default download folder. The file is named something like Module_Name_results.html. This file contains your complete results: score, all SQL you wrote, your notes, and your playground queries.
4
Send the downloaded results file to your screener. Email it, upload it, or share it however your screener has instructed.
Browser pop-up blocker: On some browsers, a print/preview window may also open in a separate tab. If your browser blocked it, your results file was still downloaded — the download always completes regardless of your pop-up settings.

10 · Tips

  • Read each question fully before writing SQL, paying attention to the Expected columns list — it tells you exactly what shape your output should be.
  • Use the playground first to explore the tables and get a feel for the data before writing your graded answers.
  • Write notes as you go — the notes area is included in your report, so you can show your thinking even if an answer does not quite work.
  • Re-run freely — there is no penalty for running a query multiple times. Iterate until you are satisfied.
  • Use the SQLite function reference if you forget the exact syntax for a function — it is built in to the right panel.
  • Column order matters for grading — make sure the columns in your SELECT match the order shown in "Expected columns".
  • Do not worry about row order — the grader sorts results before comparing, so you do not need an ORDER BY to pass.
